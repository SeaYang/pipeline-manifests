# argo-workflows搭建

基于k8s 1.23.3 安装argo workflows

## 新建namespace

```shell
kubectl create namespace argo
```

## 安装argo-workflows

### 安装命令

master节点执行

```shell
kubectl apply -n argo -f https://github.com/argoproj/argo-workflows/releases/download/v3.4.8/install.yaml
```

本仓库已经保存了[argo3.4.8.yml](./yml/argo3.4.8.yml)，也可以将该文件复制到master节点，然后安装

```shell
kubectl apply -n argo -f argo3.4.8.yaml
```

建议先把yml里面的镜像先手动拉取下来，除此之外，也手动拉取一下这个镜像：quay.io/argoproj/argoexec:v3.4.8

轩辕镜像网站里面，argo的这几个镜像不是quay.io开头的，可以docker tag改一下，比如

```shell
docker tag argoproj/argocli:v3.4.8 quay.io/argoproj/argocli:v3.4.8

docker tag argoproj/workflow-controller:v3.4.8 quay.io/argoproj/workflow-controller:v3.4.8

docker tag argoproj/argoexec:v3.4.8 quay.io/argoproj/argoexec:v3.4.8
```

## argo-workflows初体验

master节点默认不可调度，所以所有的pod都会调度到worker节点，在worker节点先手动拉取镜像：library/alpine:3.23.2，然后在master执行

```shell
root@k8s-master:/home/seayang# kubectl create -n argo -f - <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Workflow
metadata:
  generateName: test-alpine-
spec:
  entrypoint: main
  templates:
  - name: main
    container:
      image: library/alpine:3.23.2
      imagePullPolicy: Never
      command: [echo]
      args: ["Hello Argo! My pod is haha"]
EOF
```

输出

```shell
workflow.argoproj.io/test-alpine-5hm48 created
```
